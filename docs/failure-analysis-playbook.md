# Failure Analysis Playbook

| # | Symptom | Likely Cause | Artifact to Inspect | Risk Level | Safe Operator Response |
|---:|---|---|---|---|---|
| 1 | Agent changed workflow file and stalled | Approval gate required for workflow changes | PR timeline, session log | High | Pause merge, request human approval, inspect changed workflow diff |
| 2 | Downstream job cannot find expected output | `$GITHUB_OUTPUT` name mismatch or missing job output | Workflow logs, workflow YAML | Medium | Fix output key, rerun workflow, verify producing job artifact |
| 3 | MCP tool unavailable | Tool not listed or server config mismatch | Agent file, MCP config | Medium | Confirm server name/tool name, narrow allowlist, retry after review |
| 4 | Broad MCP access appears in agent profile | Wildcard allowlist such as `server/*` | `.agent.md`, MCP config | High | Replace wildcard with explicit tools, require security review |
| 5 | Agent resumed stale session | Session state older than branch changes | Session log, memory file | Medium | Reset or re-scope session, validate current branch state |
| 6 | Reviewer and builder disagree on output | Conflicting task interpretation or stale artifact | Reviewer findings, builder summary | Medium | Consolidate conflicts, require planner clarification |
| 7 | Artifact deleted before audit review | Retention or manual deletion issue | Audit log, workflow artifacts | High | Block final approval, regenerate artifact, preserve deletion record |
| 8 | Hook blocked execute access | Pre-tool policy denied command execution | Hook event log | Medium | Confirm deny was expected, escalate only if execution is approved |
| 9 | Memory fact conflicts with current repo | Stale repository fact or invalid citation | Memory files, current code | Medium | Prune stale memory, regenerate validated repository facts |
| 10 | Security gate failed due to permissions | Workflow lacks required permission or has unsafe permission | Security gate logs, workflow YAML | High | Correct permissions, rerun gate, avoid broad permissions unless justified |
| 11 | Branch scope mismatch | Agent ran on default branch or wrong PR branch | Workflow context, session log | High | Stop action, re-run on correct branch/ref, verify no unintended writes |
| 12 | Human approval skipped for high-risk action | Missing approval condition or weak workflow gate | PR timeline, workflow YAML | Critical | Revert if needed, add explicit approval gate, conduct audit |
| 13 | Final report omits security finding | Consolidator consumed wrong or stale artifact | Consolidated report, audit artifact | High | Block final decision, regenerate report from current inputs |
| 14 | Matrix job hides partial failure | One axis failed but final job ignored status | Workflow logs, matrix strategy | Medium | Add explicit needs/status checks and artifact validation |
| 15 | Agent repeats previously completed work | Missing durable state or failed resume detection | Session log, planner artifact | Low/Medium | Reconcile state, update progress artifact, resume from last verified step |
| 16 | Remote MCP endpoint used without trust notes | External tool boundary undocumented | MCP config, governance review | High | Require trust review, document data flow, narrow or disable access |
| 17 | Planner artifact missing required success criteria | Planner skipped a required field or schema validation was not enforced | `artifacts/planner-output.json`, `schemas/planner-output.schema.json` | Medium | Block execution, regenerate the planner artifact, and verify success criteria before handoff |
| 18 | Consolidated report missing unresolved security findings | Consolidator consumed stale inputs or omitted security audit findings | `artifacts/consolidated-operator-report.json`, `artifacts/security-audit-report.json` | High | Block final approval, regenerate the consolidated report from current artifacts, and require operator review |
| 19 | Reviewer note leaked into final artifact | Draft editorial note was copied into published documentation or PR output | Final docs, PR description, review artifact | Medium | Block publish or merge, remove the note, preserve evidence, and add final-render review |
| 20 | Lab answer key contradicts source artifact | Answer key was updated without rechecking the referenced artifact | Lab file, referenced artifact, README lab index | Medium | Pause release, correct the answer key, and review the affected lab against source evidence |
| 21 | Schema validates but artifact content is misleading | Required fields are present but the content omits or misstates important risk | Artifact JSON, matching schema, upstream evidence | High | Continue human review, compare against source artifacts, and regenerate misleading content |
| 22 | Downstream job consumes stale artifact name | Output or upload name changed without updating the downstream download | Workflow YAML, workflow logs, artifact list | Medium | Fix the producer/consumer name mismatch, keep gating enabled, and rerun the workflow |
| 23 | Valid JSON fails expected handoff schema | Upstream agent changed field names or omitted required handoff fields | `artifacts/planner-context-raw-malformed.json`, `schemas/agent-handoff-v2.schema.json` | High | Mark artifact invalid, block downstream invocation, and regenerate with the expected schema |
| 24 | Downstream agent processes zero files but reports success | Missing or drifted file list caused silent no-op behavior | Handoff artifact, builder summary, workflow logs | High | Treat as silent failure, block the report, and require a validated handoff with non-empty files when files are expected |
| 25 | Oversized artifact risks context-window failure | Raw logs or broad diffs exceed the downstream context budget | Handoff artifact, context budget, source logs | Medium/High | Reject oversized input, require scoped summaries with source links, and preserve unresolved risks |
| 26 | Agent repeatedly retries missing artifact download | Retry loop lacks a stop condition or explicit missing-artifact state | Workflow logs, retry annotations, artifact list | Medium | Stop after max retries, surface the missing artifact, and route to operator review |
| 27 | Final report summarizes missing context as if it were verified | Consolidator trusted incomplete or stale handoff context | Final report, upstream handoff, validation output | High | Block final approval, preserve evidence, and regenerate from validated upstream artifacts |
| 28 | Workflow succeeds but agent processed zero files | Gate checked job completion but not meaningful handoff content | Workflow logs, handoff artifact, builder summary | High | Treat as silent failure, require non-empty expected file processing, and regenerate the handoff |
| 29 | Downstream agent guesses missing artifact content | Missing artifact was summarized or inferred instead of blocked | Workflow logs, agent output, artifact list | High | Fail closed, preserve the missing-artifact evidence, and rerun only after the artifact exists |
| 30 | Risky educational example triggers active security gate failure | Gate scans inert lab examples as if they were active configuration | `mcp/`, `.github/agents/`, `.github/workflows/security-gate.yml` | Medium | Distinguish educational examples from active config, warn on examples, and fail on active risk |
| 31 | Workflow gate passes because it checks file existence but not schema validity | Artifact exists but is malformed, stale, or missing required fields | Workflow YAML, artifact JSON, schema file | High | Add schema validation before downstream agent invocation |
| 32 | Final report generated from stale artifact | Consolidator consumed prior-run evidence or unresolved risks from an old branch | Final report, artifact timestamps, commit SHA, run ID | High | Block final approval and regenerate from current branch/run artifacts |
| 33 | Hook blocks command but workflow records success | Hook denial was logged but not propagated to workflow failure state | Hook event, workflow logs, final report | High | Treat deny as blocking, fail the workflow, and preserve hook evidence |
| 34 | Context-window truncation hides unresolved risks | Oversized artifact or compressed summary dropped risk details | Context budget, source artifact, final summary | Medium/High | Reject or chunk oversized artifacts and require summaries to link to source evidence |
| 35 | Agent profile change grants execute access without review | `.agent.md` profile changed tool authority but bypassed human control-plane review | `.github/agents/`, agent profile review report, PR diff | Critical | Block merge, require human review, and verify deterministic approval gates before execution |
| 36 | Reviewer role dynamically escalates into executor role | Workflow or runner allowed role transition based on model reasoning instead of policy | Agent profiles, runner workflow, session log | High | Block the run and require explicit workflow-enforced role separation |
| 37 | Natural-language safety rule lacks deterministic enforcement | Policy exists only as prose without hook, workflow, allowlist, or approval gate | Guardrail docs, hooks, workflow policy, agent profile | High | Add deterministic enforcement before allowing high-risk tool calls |
| 38 | Agent profile consumes untrusted PR content before privileged tool use | Executor context mixed untrusted PR or issue text with high-authority operating instructions | Executor profile, PR comments, workflow logs, handoff artifacts | High | Sanitize and classify inputs, block privileged use, and preserve contamination evidence |
