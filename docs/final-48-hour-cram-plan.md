# Final 48-Hour Cram Plan

Use this realistic final review plan to strengthen repo-grounded recall without treating last-minute study as a certification guarantee.

## Day 1 - Deep Review

### Morning

- Read `README.md` for the thesis and doctrine.
- Review `docs/vulnerability-to-defense-matrix.md`.
- Review `labs/README.md` for the lab sequence.
- Read and run the validation harness in `scripts/validate_repo.py`.

### Afternoon

- Complete Labs 01-06.
- Trace each lab through its artifacts and schemas.
- Write the deterministic fail-closed control for each scenario.

### Evening

- Answer `docs/exam-style-practice-questions.md` without notes.
- Practice `docs/oral-defense-prompts.md` out loud.
- Complete `docs/weak-area-scorecard.md` and choose the lowest three areas for Day 2 review.

## Day 2 - Speed and Recall

### Morning

- Complete Labs 07-11.
- Review Zombie Pipeline diagnosis.
- Contrast schema validation with semantic validation.
- Review artifact transaction governance and provenance.

### Afternoon

- Complete `docs/hands-on-artifact-review-drills.md` under time pressure.
- Run through `docs/rapid-review-flashcards.md`.
- Rerun `python scripts/validate_repo.py` and explain each check.

### Evening

- Keep the review light.
- Write down the top 10 control patterns from memory.
- Stop cramming and preserve time for rest.

## Top 10 Control Patterns to Memorize

1. Agents propose; workflows enforce.
2. Memory is not authority.
3. Schema validity is not truth.
4. Green workflow is not valid handoff.
5. MCP is externalized authority.
6. Hooks must enforce before execution.
7. Consolidators preserve dissent.
8. Public artifacts require promotion gates.
9. Provenance binds artifacts to run, branch, commit, and job.
10. Zombie Pipelines fail closed on zero semantic progress.
