# v0.5 Capstone Review

## What v0.5 Changed

v0.5 deepened Labs 03-11 and aligned them with the existing Lab 01 and Lab 02 quality bar. The suite now moves from individual artifact literacy into MCP authority, session continuity, hook enforcement, memory governance, multi-agent consolidation, output serialization, schema-versus-semantic validation, artifact transaction integrity, and compound Zombie Pipeline failure.

## Control-Plane Doctrine

```text
Agents propose.
GitHub records.
Workflows enforce.
Artifacts prove.
Hooks constrain.
Humans approve high-risk action.
```

## What a Reviewer Should Notice

- The repo is artifact-first, not app-first.
- The labs demonstrate failure analysis, not exploit execution.
- Each lab ties visible artifacts to deeper agentic failure modes.
- The repo distinguishes advisory policy from enforceable control.
- The repo separates structural validation from semantic trust.
- The repo treats memory, tool access, workflow outputs, and final reports as state boundaries.

## What This Repo Does Not Claim

- It does not implement a production security platform.
- It does not implement real signing or attestation.
- It does not run live billing enforcement.
- It does not include runnable exploit payloads.
- It is a public educational control-plane lab.

## Recommended Review Path

1. Read `README.md`.
2. Open `docs/vulnerability-to-defense-matrix.md`.
3. Review `labs/README.md`.
4. Inspect Labs 01, 02, 05, 07, 09, and 11 as representative examples.
5. Inspect `artifacts/` and `schemas/` to see how evidence is modeled.
