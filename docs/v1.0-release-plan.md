# v1.0 Release Plan

This plan defines the remaining checks and human approval required before tagging v1.0.0.

## Release Goal

v1.0.0 should represent a stable public artifact literacy lab for agentic SDLC governance and certification cram support.

## Release Criteria

- v0.9 is merged to `main`.
- The validation harness passes on `main`.
- `README.md` is reviewed.
- `labs/README.md` is reviewed.
- `docs/vulnerability-to-defense-matrix.md` is reviewed.
- `docs/public-readiness-checklist.md` is reviewed.
- No high-risk secrets are detected.
- No unreviewed overclaims remain.
- No major broken links remain.
- The human operator approves the release tag.

## Tagging Plan

Run these commands only after the release criteria and human approval are satisfied:

```bash
git checkout main
git pull origin main
python scripts/validate_repo.py
git tag v1.0.0
git push origin v1.0.0
```

## Post-Release Use

- Use the repository for the repo-based cram sprint.
- Use `docs/final-48-hour-cram-plan.md` for final review scheduling.
- Use `docs/rapid-review-flashcards.md` and `docs/hands-on-artifact-review-drills.md` for recall and scenario diagnosis.
- Use `docs/oral-defense-prompts.md` to practice concise technical explanations.
- Cite the repository as public proof of agentic SDLC control-plane literacy, within the claims and limitations documented here.
