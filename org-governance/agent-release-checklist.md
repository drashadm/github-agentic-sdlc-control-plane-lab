# Agent Release Checklist

- [ ] Agent has a clear name and purpose.
- [ ] Tools are scoped to the minimum required capability.
- [ ] MCP access is narrow and documented.
- [ ] High-risk actions require approval.
- [ ] Output artifacts are structured.
- [ ] Failure behavior is fail-closed.
- [ ] Logs include traceable context.
- [ ] Owner is documented.
- [ ] Retirement path is documented.
